# Quality Review — Memory Vault

Weekly automated quality report for the Memory Vault domain model.

## Overview

This report is generated by `vault/quality/quality_review.py` and provides a
structured health-check of the vault across four dimensions:

| Section | What it checks |
|---|---|
| **Source Coverage** | Which source types appear in the vault, % of official vs unofficial sources, entities/decisions without source records |
| **Relation Completeness** | All relationship edges have valid `from_id`/`to_id` prefixes, valid `role` enum, valid `confidence` enum |
| **Identity Ambiguity Queue** | GitHub logins that map to multiple person entities — potential duplicates requiring manual resolution |
| **Mismatches** | Schema violations: missing `type`/`confidence` fields, unknown source types, missing `ref` in source records |

## Report Format

Each report is a dated markdown file: `YYYY-MM-DD.md`

```
## Source Coverage
## Relation Completeness
## Identity Ambiguity Queue
## Mismatches
## Summary
```

## Running

```python
from vault.quality.quality_review import write_report
from pathlib import Path

out = write_report(Path("memory/vault"))
print(out)  # Path to generated report
```

## Automation

Recommended cron: `0 8 * * 1` (Monday 08:00 BRT — after consolidation)

```
# Weekly quality report
0 8 * * 1 cd ~/.openclaw/workspace-livy-memory && \
  python3 -c "from vault.quality.quality_review import write_report; \
  write_report()" >> logs/quality-report.log 2>&1
```

## Thresholds

| Metric | Warning | Critical |
|---|---|---|
| `pct_official` | < 60% | < 40% |
| `invalid_edges` | > 0 | > 5 |
| `ambiguity_count` | > 0 | > 10 |
| `mismatch_count` | > 10 | > 30 |

## Adding Sections

To add a new quality dimension:

1. Add a `detect_<dimension>()` function in `vault/quality/quality_review.py`
2. Return `{"<dimension>_count": int, "<dimension>s": list}`
3. Add to `generate_quality_report()` and `_to_markdown()`
4. Add tests in `vault/tests/test_quality_review.py`

## File Structure

```
memory/vault/quality-review/
  README.md           — this file
  YYYY-MM-DD.md       — weekly reports (one per run)
```
